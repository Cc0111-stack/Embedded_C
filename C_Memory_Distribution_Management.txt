# C语言内存分布与管理

## 内存区域划分

C语言程序的内存空间通常划分为以下几个区域：

### 1. 代码段（Text Segment）
- 存储程序的执行代码（机器指令）
- 只读区域，防止程序意外修改指令
- 所有进程共享同一份代码（对于动态库）

### 2. 初始化数据段（Data Segment）
- 存储已初始化的全局变量和静态变量
- 例如：`int global_var = 10;`
- 程序启动时分配，程序结束时释放

### 3. 未初始化数据段（BSS Segment）
- 存储未初始化的全局变量和静态变量
- 例如：`int global_array[100];`
- 程序启动时自动初始化为零
- 名称来源于早期汇编器的"Bloc Started by Symbol"

### 4. 堆区（Heap）
- 动态内存分配区域
- 通过malloc、calloc、realloc、free等函数管理
- 生长方向：从低地址向高地址增长
- 大小受系统虚拟内存限制
- 需要程序员手动管理，容易出现内存泄漏（申请的空间不会因函数结束而被释放

### 5. 栈区（Stack）
- 存储局部变量、函数参数、返回地址等
- 由编译器自动分配和释放
- 生长方向：从高地址向低地址增长
- 速度快但容量有限
- 递归过深可能导致栈溢出

## 内存分配方式

### 静态分配
```c
int global_var = 10;          // 全局变量，存储在数据段
static int static_var = 20;   // 静态变量，存储在数据段
int global_array[100];        // 全局数组，存储在BSS段
```

### 栈分配
```c
void function() {
    int local_var = 30;       // 局部变量，存储在栈上
    int local_array[50];      // 局部数组，存储在栈上
}
```

### 堆分配
```c
#include <stdlib.h>

int *dynamic_ptr = malloc(sizeof(int) * 10);  // 分配10个整数的空间
if (dynamic_ptr != NULL) {
    // 使用动态分配的内存
    free(dynamic_ptr);  // 释放内存
    dynamic_ptr = NULL; // 避免悬空指针
}
```

## 内存管理最佳实践

### 1. malloc/calloc/realloc的使用
- malloc：分配未初始化的内存
- calloc：分配并初始化为零的内存
- realloc：调整已分配内存块的大小

### 2. 防止内存泄漏
- 每次malloc都要有对应的free
- 避免重复释放同一块内存
- 释放后将指针设为NULL

### 3. 防止缓冲区溢出
- 使用strncpy代替strcpy
- 使用snprintf代替sprintf
- 检查数组边界

### 4. 悬空指针处理
- 释放内存后立即设置指针为NULL
- 使用前检查指针是否有效

## 内存错误类型

### 1. 段错误（Segmentation Fault）
- 访问未分配的内存
- 访问权限不允许的内存（如代码段）
- 解引用空指针

### 2. 内存泄漏（Memory Leak）
- 分配的堆内存未释放
- 程序长时间运行后耗尽内存

### 3. 悬空指针（Dangling Pointer）
- 指向已释放的内存
- 使用可能导致不可预知的行为

### 4. 缓冲区溢出（Buffer Overflow）
- 向缓冲区写入超过其容量的数据
- 可能导致安全漏洞

## 调试工具

### 1. Valgrind
- 检测内存泄漏
- 检测非法内存访问
- 检测使用未初始化的内存

### 2. GDB
- 调试段错误
- 分析内存状态

### 3. AddressSanitizer (ASan)
- Google开发的内存错误检测工具
- 编译时加入-fsanitize=address选项

## 性能考虑

### 1. 减少堆分配
- 栈分配速度更快
- 对于小的、生命周期明确的变量优先使用栈

### 2. 内存池
- 对于频繁分配释放的小块内存
- 可以减少内存碎片和分配开销

### 3. 对齐
- 现代CPU对对齐的内存访问更高效
- 编译器通常自动处理对齐问题

理解C语言的内存模型对于编写高效、安全的程序至关重要，特别是在嵌入式系统和系统编程中更是如此。
